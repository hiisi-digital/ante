## Test Details

### Test Categories

- **Smoke Test**: Verifies that the package can be imported and core functions (`hasValidHeader`, `generateHeader`, `loadConfig`) work correctly.
- **Types Check**: Verifies that TypeScript type definitions are correct and can be consumed by TypeScript projects.
- **Test Suite**: Runs the full unit + integration test suite including:
  - Configuration parsing and resolution
  - Header parsing, generation, and validation
  - Contributor handling
  - Glob pattern matching
  - Column-aligned formatting
  - Full workflow integration tests (temp files, config loading, header updates)

### Runtime Notes

- **Deno**: Tests run directly against TypeScript source code
- **Node.js**: Tests run against the transpiled npm package using custom test runner
- **Bun**: Tests run against the transpiled npm package using custom test runner
- **Cross-runtime utilities**: Integration tests use a runtime abstraction layer (`tests/_utils/`) for file system operations

### Known Limitations

- E2E tests that require `Deno.Command` are excluded from Node.js/Bun testing (CLI testing is Deno-only)
- Integration tests use cross-runtime file system utilities and should work on all runtimes
- Some older runtime versions may have limited ES2022 support
